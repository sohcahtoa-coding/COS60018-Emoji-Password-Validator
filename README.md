# COS60018 Emoji Password Validator
This program prompts the user to create a password meeting strict criteria, including a minimum length, at least one number, at least one standard symbol, and at least one emoji. Because complex emojis are not single characters, the tool will only accept emojis from a hardcoded whitelist of 50 safe, 'atomic' emojis. It will reject complex ones that use invisible joiners or skin tone modifiers.

Once a valid password is provided, the program will calculate its cryptographic strength. It will take into account the expanded character pool and use standard entropy maths to figure out the total possible combinations. Finally, it will output a clear estimate of exactly how long a high-end PC would take to crack the password by brute-force (100 billion tries per second). This calculated cryptographic strength should be seen as an "upper limit". To be accurate each password character would need to be randomly generated from the pool with no enforced minimums. Also as humans we tend to add recognisable patterns to our passwords which generally reduces the strength.

The minimum password length is set to just six characters. This was a deliberate choice to prove a point about how we calculate password strength. By adding 50 emojis into the mix, the total pool of available characters expands massively. This mathematical jump in entropy shows that you can build highly secure, brute-force resistant passwords without forcing people to remember ridiculously long strings. It is all about balancing strong security with actual human usability.
